# ⚙️ 自动化命令与脚本集成

> **难度**: ⭐⭐⭐ 进阶 | **预计阅读**: 4 分钟

---

## 📋 本章要点

- OpenClaw CLI 覆盖服务管理、配置读写、会话操作、诊断排错等完整功能
- Hook 钩子系统在会话生命周期节点自动执行逻辑（如加载上下文、记录命令、保存记忆）
- Cron 定时任务在隔离 Session 中自动运行，执行记录以 JSONL 格式保存
- Agent 可调用系统命令、本地脚本、内置工具（web_fetch）和 MCP 外部服务
- 支持 Shell 补全（Bash/Zsh/Fish/PowerShell）
- Cron 表达式格式：分钟 小时 日 月 星期

## 🔑 核心知识

OpenClaw 的自动化能力建立在三大机制上：**CLI 命令行**（日常管理与脚本调用）、**Hook 钩子**（事件驱动的自动化扩展）和 **Cron 定时任务**（周期性无人值守执行）。CLI 常用命令包括 `openclaw status`（服务状态）、`openclaw config get/set`（配置读写）、`openclaw logs --follow`（实时日志）。内置 Hook 有 `boot-md`（加载 workspace 上下文）、`command-logger`（审计日志）、`session-memory`（会话记忆持久化）等。

Cron 任务在 `~/.openclaw/cron/jobs.json` 中配置，每个任务包含名称、schedule（Cron 表达式）、timezone 和 action。每次执行在独立 Session 中运行，执行记录保存在 `cron/runs/` 目录。脚本集成方面，Agent 可直接运行 Python/Bash/Node.js 脚本，内置 `web_fetch` 支持网页抓取，McPorter 可连接任意 MCP 工具服务器。

## 💻 关键命令/代码

```bash
# 核心管理命令
openclaw status && openclaw doctor
openclaw config set gateway.port 18789
openclaw logs --follow

# 查看 Cron 任务与执行记录
cat ~/.openclaw/cron/jobs.json | python3 -m json.tool
ls -lt ~/.openclaw/cron/runs/ | head -10
```
