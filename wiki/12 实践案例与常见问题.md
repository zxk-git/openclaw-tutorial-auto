# 📋 实践案例与常见问题

> **难度**: ⭐⭐⭐ 进阶 | **预计阅读**: 4 分钟

---

## 📋 本章要点

- 案例一：个人知识助手——Cron 定时搜索 + AI 摘要 + Memory 存储 + 飞书日报
- 案例二：项目监控机器人——GitHub Skills + Cron 巡查 + 飞书群推送
- 案例三：教程自动生成——complex-task-automator Skill 驱动多章节批量生成
- Gateway 常驻内存约 100-200MB，每个 Agent 会话约 50-100MB
- 支持切换 AI 模型（OpenAI / Claude / 本地模型），在 openclaw.json 中配置
- 国内网络建议配置 HTTP 代理（`proxy.http` / `proxy.https`）

## 🔑 核心知识

三个案例代表了 OpenClaw 的典型应用场景。**个人知识助手**用 tavily-search + memory + feishu 三个 Skills，通过 Cron 每天 3 次自动搜索热门话题、生成摘要、分类存储并推送飞书日报，月积累 500+ 条目。**项目监控机器人**用 github + feishu Skills，工作日每天 2 次自动检查 PR/Issue/CI 状态并推送到飞书群，PR 超 3 天未合并或 CI 失败自动告警。

**教程自动生成**是最复杂的案例（即本教程的生成方式），使用 `complex-task-automator` Skill，配合 write_chapter.py（章节生成）、analyze_progress.py（进度追踪）、git_workflow.py（自动提交）等脚本，13 章约 4 小时全部自动完成，平均质量分 93+。常见问题方面：系统资源占用低（空闲时 CPU ≈ 0）；模型切换改 `ai.provider` + `ai.model`；国内网络需配置代理。

## 💻 关键命令/代码

```bash
# 验证 Skills 已安装
ls ~/.openclaw/workspace/skills/tavily-search/SKILL.md

```
