# 🔒 安全与权限管理

> **难度**: ⭐⭐⭐ 进阶 | **预计阅读**: 22 分钟

---

## 📋 本章要点

- OpenClaw 采用**纵深防御**策略，在设备认证、凭证管理、执行控制、网络安全、审计日志五层构建安全屏障
- 设备标识存储于 `~/.openclaw/identity/`，包含唯一 `deviceId` 和认证 Token
- 凭证目录 `~/.openclaw/credentials/` 实现文件级、权限级、环境级、时间级四维隔离
- 权限分为 4 级：Level 0 只读 → Level 1 受限写入 → Level 2 命令执行 → Level 3 系统管理
- `exec-approvals.json` 控制 Agent 执行命令的审批策略（allow / deny / prompt）
- Skill 沙箱隔离通过 `_meta.json` 权限声明实现最小权限原则
- 支持 AES-256-GCM 加密存储凭证，以及 Vault 集成方案
- 安全审计日志 `config-audit.jsonl` 记录所有操作，支持告警规则配置

## 🔑 核心知识

OpenClaw 安全模型的核心是"最小权限、纵深防御"。每个 Agent 实例通过设备指纹唯一标识，凭证按服务隔离存储，文件权限默认 `600/700`。执行审批机制是安全核心——Agent 运行命令前，系统根据 `exec-approvals.json` 中的正则规则决定 allow（放行）、deny（拒绝）或 prompt（交互审批）。

Skill 权限声明在 `_meta.json` 中定义，支持 `minimal`、`standard`、`elevated` 三种沙箱模式，确保 Skill A 无法越权读取 Skill B 的凭证。网络层面支持 TLS、IP 白名单和速率限制。所有操作记录在 `config-audit.jsonl`，可配置告警规则实现安全事件实时通知。

## 💻 关键命令/代码

```bash
# 查看设备身份
cat ~/.openclaw/identity/device.json

# 添加凭证
openclaw credential add github-token --type token --value "ghp_xxx" --expires "2026-12-31"

# 加密所有凭证
openclaw credential encrypt --all
```
