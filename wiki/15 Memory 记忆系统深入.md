# 🧠 Memory 记忆系统深入

> **难度**: ⭐⭐⭐ 进阶 | **预计阅读**: 25 分钟

---

## 📋 本章要点

- 记忆分为三种类型：**工作记忆**（会话级）、**短期记忆**（天/周级）、**长期记忆**（永久）
- 所有持久化记忆以 Markdown 文件存储在 `~/.openclaw/workspace/memory/` 目录
- 记忆文件遵循统一格式：标题 + 元数据（分类/标签/重要性）+ 结构化内容
- `MEMORY.md` 是全局索引文件，Agent 启动时首先读取以获取记忆概览
- 文件命名规范：`YYYY-MM-DD-topic.md`，支持分类体系和标签元数据
- 记忆检索支持三种策略：语义匹配、关键词检索、时间衰减模型
- 记忆合并与清理机制解决记忆膨胀问题，支持自动清理规则
- 记忆在 Agent 决策中通过 Prompt 注入影响推理行为

## 🔑 核心知识

Memory 系统借鉴认知科学，让 Agent 具备"经验积累"能力。工作记忆随会话结束清除，Agent 会自动评估哪些内容值得固化为短期或长期记忆。记忆文件使用 Markdown 引用块存储元数据（创建时间、分类、标签、重要性、状态），便于 Agent 解析和人类阅读。

记忆生命周期为：创建 → 活跃 → 冷却 → 归档/删除。检索引擎综合语义匹配、关键词搜索和时间衰减三种策略对记忆排序。当记忆文件过多时，可通过合并策略（相同主题合并、时间段聚合）和自动清理规则（过期删除、低重要性归档）控制规模。记忆注入 Prompt 后，直接影响 Agent 的行为模式和决策质量。

## 💻 关键命令/代码

```bash
# 查看记忆目录
ls -la ~/.openclaw/workspace/memory/

# 查看记忆索引
cat ~/.openclaw/workspace/MEMORY.md

# 记忆文件命名示例
# 2026-03-06-security-config.md
```
