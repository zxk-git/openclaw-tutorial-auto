# 📖 术语表

> OpenClaw 核心概念与术语解释

---

| 术语 | 英文 | 说明 |
|------|------|------|
| **Agent** | Agent | OpenClaw 的核心运行单元，负责接收指令并执行任务。每个 Agent 独立运行，拥有身份和配置。 |
| **Gateway** | Gateway | 消息网关，负责接收外部请求（飞书、HTTP 等）并路由到对应的 Agent。 |
| **Skill** | Skill | 可复用的功能插件，扩展 Agent 的能力。类似于 ChatGPT 的 Plugin。 |
| **ClawdHub** | ClawdHub | OpenClaw 的技能市场平台，用于发布、安装和分享 Skills。 |
| **SKILL.md** | SKILL.md | Skill 的说明文档，定义技能用途、参数和使用方式，供 Agent 阅读理解。 |
| **_meta.json** | _meta.json | Skill 的元数据文件，定义版本、依赖、作者等结构化信息。 |
| **Cron Job** | Cron Job | 定时任务，通过 `openclaw cron` 管理，支持 cron 表达式。 |
| **Credential** | Credential | 凭证管理，安全存储 API key、token 等敏感信息。 |
| **Device** | Device | 运行 OpenClaw 的物理或虚拟主机，通过设备身份认证。 |
| **Pairing** | Pairing | 设备配对，将外部客户端（如飞书群）与 Agent 关联。 |
| **Session** | Session | Agent 的一次对话会话，保持上下文状态。 |
| **Memory** | Memory | Agent 的持久化记忆系统，跨会话保存重要信息。 |
| **Workspace** | Workspace | Agent 的工作空间，存放配置、记忆、技能等文件。 |
| **Delivery Queue** | Delivery Queue | 消息投递队列，确保消息可靠送达。 |
| **Browser Relay** | Browser Relay | 浏览器中继服务，允许 Agent 操控浏览器完成网页任务。 |
| **Canvas** | Canvas | 可视化画布，展示 Agent 的工作状态和产出。 |

```
           │

│  └─────────┘ └────────────┘  │
│  ┌─────────┐ ┌────────────┐  │
│  │  Cron   │ │ Credential │  │

└──────────────────────────────┘
```
