# 💬 飞书集成与消息自动化

> **难度**: ⭐⭐⭐ 进阶 | **预计阅读**: 3 分钟

---

## 📋 本章要点

- 飞书通道通过 WebSocket 实现实时双向通信，无需公网回调地址
- 需在飞书开放平台创建企业自建应用，开通机器人能力和相关权限
- 支持私聊对话（DM）、群聊 @提及响应、文件/图片处理、Emoji 回应
- 私聊策略：open（全部响应）、allowlist（白名单）、paired（已配对设备）
- 结合 Cron 实现自动定时推送（日报、周报、监控告警）
- 通过 `openclaw message send` 可主动向用户/群组发消息
- 安全机制包括访问白名单、配对认证和 IP 白名单

## 🔑 核心知识

飞书集成的架构为：飞书 App ←WebSocket→ OpenClaw Gateway → Agent。配置流程分四步：①在飞书开放平台创建应用获取 App ID 和 App Secret；②开通 `im:message`、`im:resource` 等权限并选择 WebSocket 模式；③在 `openclaw.json` 的 `channels.feishu` 中填入凭证；④启动 Gateway 验证连接。

消息自动化的核心模式是 Cron 定时触发 → Agent 执行任务 → 生成报告 → 飞书通道发送。生产环境安全最佳实践：使用 `allowlist` 策略限制可交互用户、App Secret 通过环境变量注入而非硬编码、定期审查已配对设备列表。

## 💻 关键命令/代码

```bash
# 验证飞书连接
openclaw gateway start
openclaw gateway status

# 主动发送消息
openclaw message send \
  --channel feishu \
  --target "<open_id>" \
  --message "今日任务已完成"
```
