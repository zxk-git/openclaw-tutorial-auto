# 🔍 故障排查与日志分析

> **难度**: ⭐⭐ 中级 | **预计阅读**: 4 分钟

---

## 📋 本章要点

- `openclaw doctor` 一键诊断环境/配置/服务/网络，`--fix` 可自动修复
- 日志体系包含 Gateway 主日志、配置审计日志（JSONL）、Agent 会话日志、Cron 日志
- 日志级别从 DEBUG 到 FATAL 共 5 级，可通过 `openclaw config set logLevel` 调整
- 四大常见故障：Gateway 无法启动、Agent 无响应、Skill 加载失败、Cron 不执行
- 性能优化涵盖 Gateway 内存、响应速度和磁盘清理三个方向
- 手动排查清单：进程状态 → 端口占用 → 配置文件语法 → 磁盘空间

## 🔑 核心知识

故障排查的首选工具是 `openclaw doctor`，它覆盖 Node.js 版本、Gateway 进程、配置语法、Skills 加载、通道连接、磁盘空间和网络连通性七大检查项，部分问题（如 Gateway 停机、配置损坏、通道断开）支持自动修复。排查时可加 `--json` 参数以便脚本调用。

日志分析的核心手段：`openclaw logs --follow` 实时跟踪 Gateway 日志，`journalctl --user -u openclaw-gateway` 查看 systemd 日志（支持按级别和时间范围过滤），`config-audit.jsonl` 记录所有配置变更审计。常见故障的排查路径——Gateway 无法启动时先查端口占用再检查配置语法；Agent 无响应时检查 Gateway 连接状态和 AI 模型可用性；Skill 加载失败时验证 SKILL.md 的 YAML frontmatter 格式。

## 💻 关键命令/代码

```bash
# 一键诊断
openclaw doctor --fix

# 实时日志 + 按级别过滤
openclaw logs --follow
journalctl --user -u openclaw-gateway --priority=err --since "1 hour ago"

# 手动检查端口与配置
lsof -i :18789
python3 -m json.tool ~/.openclaw/openclaw.json
```
