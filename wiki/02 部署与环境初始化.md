# 🔧 部署与环境初始化

> **难度**: ⭐ 入门 | **预计阅读**: 3 分钟

---

## 📋 本章要点

- 支持本地单机、多 Agent、Docker 容器、分布式四种部署模式
- Gateway 是核心网关服务，通过 `openclaw daemon` 命令管理
- `openclaw.json` 包含 gateway、channels、skills、plugins 等核心配置项
- Linux 使用 systemd 管理 Gateway 服务，支持开机自启
- Docker 部署支持官方镜像和 docker-compose
- 环境变量优先级高于配置文件，适合 CI/CD 场景
- `openclaw doctor` 可一键诊断并自动修复常见问题
- 支持 Azure、DigitalOcean、腾讯云等主流云平台部署

## 🔑 核心知识

Gateway 是 OpenClaw 的消息路由和 Agent 调度中枢，默认监听 18789 端口。部署时有两种管理方式：`openclaw daemon`（注册为系统服务，推荐生产环境）和 `openclaw gateway start`（直接启动，适合调试）。配置文件 `openclaw.json` 中最重要的是 `gateway`（端口/绑定/认证）、`channels`（飞书等消息通道）和 `skills.entries`（技能与 API Key）三大块。

环境变量（如 `OPENCLAW_GATEWAY_PORT`、`OPENAI_API_KEY`）可覆盖配置文件设置，在容器化和 CI/CD 场景尤为实用。部署完成后务必运行 `openclaw doctor` 全面检测依赖、配置、端口和网络连通性，加上 `--fix` 参数可自动修复已知问题。

## 💻 关键命令/代码

```bash
# 安装 Gateway 守护进程
openclaw daemon install --port 18789 --token <your-token>

# systemd 服务管理（Linux）
systemctl --user status openclaw-gateway
systemctl --user enable openclaw-gateway

# 健康诊断与自动修复
openclaw doctor --fix
```
