# 🔌 高级场景：第三方平台集成

> **难度**: ⭐⭐⭐⭐ 高级 | **预计阅读**: 5 分钟

---

## 📋 本章要点

- 两种集成方式：Skill 封装（封装 CLI/API，单向调用）和 MCP Server（协议连接，双向通信）
- Google Workspace 通过 `gog` Skill 集成 Gmail、Calendar、Drive、Docs、Sheets
- GitHub 通过 `github` Skill 封装 `gh` CLI，管理 Issue、PR、CI/CD
- 飞书既有内置消息通道，也可通过 MCP 获得更丰富的 API 能力
- McPorter 可接入任何支持 MCP 协议的自定义服务
- 所有集成遵循四步模式：安装 → 认证 → 权限声明 → 测试验证
- 调试排障步骤：认证状态 → API 连通 → 权限范围 → 日志分析

## 🔑 核心知识

OpenClaw 通过 Skill 封装和 MCP Server 两种方式实现第三方集成。Skill 封装适合已有成熟 CLI 的服务（如 GitHub 的 `gh`、Google 的 `gog`），开发门槛低；MCP Server 适合需要实时双向通信的场景（如飞书深度 API、Notion），通过 McPorter 管理配置。

Google Workspace 集成（`gog` Skill）覆盖 Gmail 搜索/发送、Calendar 日程管理、Drive 文件操作和 Sheets 数据读写，需先通过 OAuth 完成认证。GitHub 集成通过 `gh auth login` 或 Token 认证后，可在 Agent 中直接执行 Issue/PR 管理和 CI 状态检查。自定义 MCP 集成通过 `config/mcporter.json` 声明 Server 启动命令和环境变量。集成调试时按"认证→连通→权限→日志"四步排查。

## 💻 关键命令/代码

```bash
# Google Workspace
gog auth login
gog gmail search "from:boss after:2026/03/01"
gog calendar events --days 7

# GitHub
gh auth login
gh issue list --repo owner/repo --state open
gh pr create --title "feat: xxx" --body "说明"

# MCP 管理
cat ~/.openclaw/workspace/config/mcporter.json | python3 -m json.tool
```
