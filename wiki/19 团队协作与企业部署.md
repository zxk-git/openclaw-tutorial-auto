# 👥 团队协作与企业部署

> **难度**: ⭐⭐⭐ 进阶 | **预计阅读**: 22 分钟

---

## 📋 本章要点

- 三种团队使用模式：**共享 Agent**（2-5人）、**个人 Agent**（5-20人）、**混合模式**（10-100人）
- RBAC 权限模型：super_admin / admin / developer / user / guest 五种角色
- 团队知识库支持共享 + 私有分层架构，可配置同步策略
- 多级审批工作流：支持审批场景定义、审批链和审计记录
- 企业安全合规：数据保护、完整审计日志、备份与灾难恢复方案
- 私有化部署：离线安装、私有模型接入、网络隔离方案
- 团队最佳实践：规范与标准文档、培训计划、新成员入职清单
- 混合模式推荐：1-2 个共享 Agent 处理通用任务 + 核心成员个人 Agent

## 🔑 核心知识

从个人工具扩展到团队环境时，核心挑战是多用户权限和数据隔离。共享模式下所有用户对话记忆互相可见，适合小团队快速上手；个人模式每人独立 Agent 实例和私有存储，隐私性最强但资源消耗高；混合模式兼顾两者优势，是大多数团队的推荐选择。

RBAC 权限模型精细控制每个角色的操作范围——guest 只能使用指定 Agent 且每日消息限额 50 条，developer 可开发部署 Skill，admin 可管理用户和审批流程。企业部署还需关注私有化方案（离线安装、私有模型接入、网络隔离），以及完整的审计日志和灾难恢复能力。团队入职清单确保新成员快速上手。
