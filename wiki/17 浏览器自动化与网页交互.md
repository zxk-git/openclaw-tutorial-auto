# 🌐 浏览器自动化与网页交互

> **难度**: ⭐⭐⭐ 进阶 | **预计阅读**: 22 分钟

---

## 📋 本章要点

- OpenClaw 采用 **Agent → Browser Relay → Chromium** 三层架构实现浏览器自动化
- Browser Relay 是中继服务，统一管理浏览器实例生命周期、请求队列和速率控制
- 浏览器用户数据（Cookie、LocalStorage）持久化在 `~/.openclaw/browser/openclaw/user-data/`
- 支持网页浏览、内容提取（文本/结构化数据）、表单填写、按钮点击、文件上传下载
- 截图功能结合 Vision 模型实现页面视觉分析与变化监控
- 内置速率限制（默认 30 req/min）和 robots.txt 遵守机制，确保合规使用
- 与 Puppeteer MCP 的区别：Browser Relay 开箱即用、会话保持、内置限流和缓存
- 可与 Skill 结合构建浏览器驱动的自动化任务（竞品监控、招聘采集等）

## 🔑 核心知识

Browser Relay 将浏览器操作与 Agent 推理解耦。Agent 通过 HTTP/WebSocket 向 Relay 发送指令，Relay 管理请求队列、速率限制和页面缓存，再通过 CDP（Chrome DevTools Protocol）控制 Chromium 执行实际操作。这种架构支持最多 5 个并发页面，缓存命中率可达 62%。

配置文件 `~/.openclaw/browser-relay/config.json` 控制浏览器启动参数、Relay 端口、速率限制和缓存策略。浏览器 Profile 持久化意味着登录态在重启后保留。合规方面，系统默认遵守 robots.txt，请求间隔不低于 2 秒，并提供法律与伦理使用指南。截图存储在 `~/.openclaw/media/inbound/`，支持全页和区域截图。

## 💻 关键命令/代码

```bash
# 启动 Browser Relay
openclaw browser start

# 查看状态
openclaw browser status

# 清理缓存（保留登录态）
openclaw browser clear-cache
```
